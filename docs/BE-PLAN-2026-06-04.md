### P2 — `agent_prompts` rows for edit/proofread/qa phases

**Status: not started.**

`/api/mac-rag/generate` writes a `prompt_edits` audit row only when an
active `agent_prompts` row exists for the phase (global, `user_id IS NULL`).
Currently no such rows exist for `edit`, `proofread`, or `qa` — prompt-edit
audit trail is empty for those phases.

**Fix scope:** seed 3 rows in `agent_prompts` table (`phase: 'edit'`,
`phase: 'proofread'`, `phase: 'qa'`) with the current default prompts from
`lib/agents/phase-prompts.ts`. Can be done via a migration or a one-shot
admin script.

**Deliverable:** new migration `007_seed_agent_prompts.sql` OR
`scripts/seed-agent-prompts.ts`.

### P3 — `edit_pattern` before/after phrase extraction from diff

**Status: partial.** `EditPatternModal` collects before/after text when the
user explicitly fills it; skip path passes `edit_pattern: null`. The current
design requires the user to *manually type* the before/after phrase.

**Enhancement (optional):** auto-suggest before/after by diffing the accepted
`proposed_text` against the segment's current `target_text`. The suggestion
would pre-fill the modal's input fields; the user can edit or clear them.

**Deliverable:** a ~20-line diff utility in `lib/utils/diff-suggest.ts` +
wiring into `SuggestionPanel`'s `handleAcceptClick` to pass pre-filled values
to `EditPatternModal`.

---

### P4 — StyleRuleModal `scope_ref` for document-level scoping

**Status: partial.** `scope='article'` uses `articleId` (wired). `scope='document'`
currently sets `scope_ref = null` (RPC accepts null gracefully, but the rule is
not anchored to any document).

**Fix scope:** `SuggestionPanel` already receives `articleId`. The article's
`id` is a UUID that corresponds to a document context. For true document-level
scoping, we'd need a `documentId` (which may be the same as `articleId` in the
current data model — verify). If equivalent, `scope='document'` can also use
`articleId` as `scope_ref`.

**Deliverable:** one-line change in `StyleRuleModal`'s `handleSave()` to use
`scope_ref = articleId` for `scope='document'` as well (with a comment
explaining the equivalence), or define a proper document concept if they differ.
