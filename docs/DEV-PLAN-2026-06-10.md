1. Import GitHub repo into Vercel.

### 3.4 GDrive OAuth for production

**Current state:** `/api/pdfs/[articleId]` uses a developer token. This must change for production.

Steps:
1. Create a GCP service account in the existing GCP project.
2. Enable the Google Drive API.
3. Share the `kendo-translation-pdfs` GDrive folder with the service account email (Reader role).
4. Download the service account JSON key.
5. Set `GDRIVE_SERVICE_ACCOUNT_JSON` in production env (contents of the JSON file, or base64-encoded).
6. Update `app/api/pdfs/[articleId]/route.ts`: detect `gdrive:` prefix → use service-account auth to call the Drive Files API.
7. Smoke-test: `GET /api/pdfs/<articleId>` for an article with `paired_pdf_path=gdrive:<fileId>` must return PDF bytes.

### 3.6 CI/CD pipeline skeleton

Create `.github/workflows/ci.yml`:

```yaml
name: CI
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  typecheck:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '20' }
      - run: npm ci
      - run: npx tsc --noEmit

  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '20' }
      - run: npm ci
      - run: npx next lint

  test:
    needs: [typecheck, lint]
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '20' }
      - run: npm ci
      - run: npx playwright install --with-deps firefox
      - run: |
          nohup npx next dev --turbopack --port 3001 > /tmp/next-dev.log 2>&1 &
          sleep 15
      - run: npm test
        env:
          TEST_BASE_URL: http://localhost:3001
          NEXT_PUBLIC_SUPABASE_URL: ${{ secrets.NEXT_PUBLIC_SUPABASE_URL }}
          NEXT_PUBLIC_SUPABASE_ANON_KEY: ${{ secrets.NEXT_PUBLIC_SUPABASE_ANON_KEY }}
          SUPABASE_SERVICE_ROLE_KEY: ${{ secrets.SUPABASE_SERVICE_ROLE_KEY }}

  deploy:
    needs: [test]
    if: github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: amondnet/vercel-action@v25
        with:
          vercel-token: ${{ secrets.VERCEL_TOKEN }}
          vercel-org-id: ${{ secrets.VERCEL_ORG_ID }}
          vercel-project-id: ${{ secrets.VERCEL_PROJECT_ID }}
          vercel-args: '--prod'
```

> **Note:** Playwright tests write real data to Supabase. Use a dedicated **staging** Supabase project for CI to avoid polluting production data. Store staging credentials as GitHub secrets separate from production.
