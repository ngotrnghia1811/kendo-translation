## 5. Code: implementation gaps the docs reference

These are the gaps cited as `[GAP]` throughout `docs/MAC-RAG-EXAMPLES.md`.
They are the work items the example doc points at without implementing.

### Memory & retrieval

- **Phase 4b Memory Update**: entirely missing across all four tasks. No
  endpoint, no UI, no `lib/learning/`.
- **Domain Corpus** retrieval source: not implemented.
- **Cross-Lingual KB** retrieval source: not implemented.
- **`style_guide` table**: does not exist; style rules live in prompts.
- **`qa_issue_patterns` view**: does not exist; QA Phase 1 retrieval has
  no historical issue source.
- **TM retrieval**: uses Levenshtein + Jaccard + n-gram, not embeddings.
  Default `minMatchScore: 50`.
- **Terminology retrieval**: no confidence-threshold filter.

### Pipeline machinery

- **Edit MAC-RAG-ification**: edit currently goes through
  `/api/agents/[phase]` (one LLM call, no retrieval, no candidates, no
  scoring). The generalized design subsumes this; not yet implemented.
- **Proofread MAC-RAG-ification**: same as edit.
- **QA pipeline**: no QA agent exists at all today. Walkthrough is
  pure design.
- **Unified `/api/mac-rag` route**: not yet built; today the orchestrator
  is `/api/translate/mac-rag`.
- **`lib/mac-rag/tasks/` registry**: not yet built.

- **Candidates Panel**: partial (translate only). Edit and proofread

- **Post-Production Panel**: does not exist.
- **Composed-prompt review/edit UI**: does not exist (this is TODO 2).

### Auto-accept machinery

- **24-hour revert window for auto-accepted suggestions**: design only.

## 11. Quick-reference invariants (don't forget)

- Cooperation-first: agents propose, humans accept. No agent ever writes
  `segments.target_text` directly.
- Auto-accept is opt-in per document, proofread-only, and never bypasses
  the audit trail.
- Phase advance is always human, even when content change was
  auto-accepted.
- QA-advisory writes no `segment_suggestions` row; `qa_issues` rows are
  written only at the moment of human triage.
- `qa_approved` is terminal.
- Commit style: imperative `<type>(<scope>): <subject>`, multi-paragraph
  body ending with test-count delta, real wall-clock timestamps.
- Push to `origin` always requires explicit user authorization.
- Soft-lock primitive, not CRDT.
- Global roles: `{admin, translator, reader}`. Per-document, per-phase
  capabilities granted via assignments.
